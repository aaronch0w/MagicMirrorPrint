# Magic Mirror Print

An Android sideload app that watches for new client skin analysis reports generated by the **Magic Mirror** medspa device and automatically prints them to the network printer — without staff ever leaving the Magic Mirror app or touching the SD card.

---

## The Problem

The Magic Mirror device runs a proprietary Android app that performs multi-camera facial skin analysis. When a scan completes, staff tap "Save to SD Card" which writes a PDF report. To print it, they previously had to:

1. Exit the Magic Mirror app
2. Open the file explorer
3. Navigate to the SD card
4. Find the client's folder
5. Print the PDF

This was too slow and disruptive when a client is sitting in the chair.

## The Solution

Magic Mirror Print runs silently as a **foreground background service**, watching the SD card directory tree for new reports. The moment a new `customer_report.pdf` is written:

- **Auto-print mode (default):** The report is sent to the printer immediately. A notification sound plays so staff know it's on its way. No tapping required.
- **Manual mode:** A heads-up notification appears with **Print** and **Dismiss** buttons. Staff tap Print and move on.

The service starts automatically on device boot and restarts itself if it was previously running — no daily setup needed.

> **Note:** The Magic Mirror app runs in Android kiosk/lock-task mode, which blocks overlay notifications from other apps. Auto-print mode is recommended for this device.

## SD Card Directory Structure

The Magic Mirror app writes reports in this exact structure:

```
<root>/                              ← configurable, typically /storage/sdcard1/report/
  └── <Client Name>/                 ← folder named after client entered at scan time
        └── 2026_06_27_23_03_33/    ← date_time folder: YYYY_MM_DD_HH_MM_SS
              ├── img/               ← facial scan images (not our concern)
              └── pdf/
                    └── customer_report.pdf   ← THIS is what we watch for and print
```

**Key behaviors:**
- Each new client gets a new folder under root (named by staff at scan time)
- Each scan session gets a new date/time subfolder under the client folder
- The target file is always named `customer_report.pdf` (configurable in Settings)
- The `CLOSE_WRITE` FileObserver event is used — fires when the writing process closes the file, ensuring the PDF is fully written before printing

## Architecture

### Files

```
app/src/main/java/com/magicmirrorprint/
├── WatcherService.kt        Foreground service + dynamic recursive FileObserver tree
├── NotificationHelper.kt    Persistent service notification + report alert + sound
├── PrintManager.kt          IPP/1.1 print over HTTP on port 631 (no print dialog)
├── OverlayManager.kt        TYPE_APPLICATION_OVERLAY window for manual-mode alert
├── AppPrefs.kt              SharedPreferences wrapper — typed getters/setters for all settings
├── SettingsActivity.kt      Main UI: all settings fields, folder picker, start/stop service
├── FolderPickerActivity.kt  Filesystem browser so staff can tap to the report folder at setup
└── BootReceiver.kt          BOOT_COMPLETED receiver — auto-restarts service after device reboot
```

### How the FileObserver tree works

Android's `FileObserver` does not recurse into subdirectories natively. We solve this with a **dynamic observer tree**:

1. **Root observer** watches `<root>/` for new client directories → attaches a client observer
2. **Client observer** watches `<ClientName>/` for new date/time directories → attaches a date observer
3. **Date observer** watches `YYYY_MM_DD_HH_MM_SS/` for the `pdf/` subdirectory → attaches a PDF dir observer
4. **PDF dir observer** watches `pdf/` for `CLOSE_WRITE` on `customer_report.pdf` → triggers print

All observers are held in `activeObservers: MutableList<FileObserver>` to prevent garbage collection. Pre-existing folders at service start are also observed (handles the case where the service restarts mid-day with clients already scanned).

### Printing (IPP over HTTP)

Uses **IPP/1.1 (Internet Printing Protocol)** transported over HTTP POST on port 631 — no Android Print Framework dialog, no user interaction. Constructs a minimal `Print-Job` IPP request:

- `attributes-charset`: utf-8
- `attributes-natural-language`: en
- `printer-uri`: `ipp://<ip>:<port>/ipp/print`
- `document-format`: application/pdf
- `media`: `na_letter_8.5x11in` — forces US Letter regardless of document page size
- `print-scaling`: `fit` — scales A4 content (from Chinese device) to fill Letter paper

Runs on a background thread. Compatible with any network printer supporting IPP or AirPrint (HP, Epson, Canon, Brother, etc.).

### Print flow

**Auto-print mode (default):**
```
New customer_report.pdf detected
        │
        ▼
  IPP print job sent to printer
        +
  Notification sound plays
        +
  Quiet "Printing: [Client]" status notification
```

**Manual mode:**
```
New customer_report.pdf detected
        │
        ▼
┌─────────────────────────────────────────┐
│  New Report Ready                       │
│  Client: Sarah Johnson                  │
│  Scan: Jun 27, 2026  11:03 PM          │
│                                         │
│  [Print]              [Dismiss]         │
└─────────────────────────────────────────┘
        │                    │
        ▼                    ▼
  IPP print job         Notification
  sent to printer         cleared
```

Two `NotificationChannel`s:
- `channel_watcher` — IMPORTANCE_LOW — persistent "Magic Mirror Print Active" indicator
- `channel_report` — IMPORTANCE_HIGH — report alert (manual mode)

## Permissions

```xml
READ_EXTERNAL_STORAGE     <!-- SD card access — no MANAGE_EXTERNAL_STORAGE needed on API 27 -->
WRITE_EXTERNAL_STORAGE    <!-- Needed for some SD card paths on Android 8 -->
FOREGROUND_SERVICE        <!-- Required to run persistent background service on Android 8+ -->
INTERNET                  <!-- IPP network printing -->
RECEIVE_BOOT_COMPLETED    <!-- Auto-restart service on device reboot -->
SYSTEM_ALERT_WINDOW       <!-- Draw-over-other-apps for manual-mode overlay (user must grant) -->
```

`SYSTEM_ALERT_WINDOW` is only needed for manual mode. The app shows a banner with a **Grant Permission** button if it isn't granted yet.

## Settings

All settings persist in SharedPreferences across restarts.

| Setting | Default | Description |
|---|---|---|
| Root Folder | *(must set)* | Absolute path to the Magic Mirror report directory on SD card. Use the in-app folder browser. |
| Target Filename | `customer_report.pdf` | Filename to watch for. Configurable in case a Magic Mirror update changes it. |
| Printer IP | *(must set)* | Local network IP of the target printer (e.g. `10.0.0.101`) |
| Printer Port | `631` | IPP port — 631 is standard, rarely needs changing |
| Notification Timeout | `60` seconds | How long the manual-mode notification stays before auto-dismissing |
| Auto-print | **ON** | ON: prints immediately + plays sound. OFF: shows Print/Dismiss notification (manual mode). |

## Build & Deploy

### Build environment

Builds from the command line — no Android Studio required.

**Requirements:**
- JDK 17+ (`sudo apt-get install openjdk-17-jdk`)
- Android SDK with `platforms;android-33` and `build-tools;33.0.2`
- `ANDROID_HOME` and `JAVA_HOME` set in environment

**Build:**
```bash
cd MagicMirrorPrint
./gradlew assembleDebug
# APK output: app/build/outputs/apk/debug/app-debug.apk
```

### WiFi ADB deployment

The Magic Mirror device does not expose ADB over the network by default. To enable it:

1. On the device: Settings → About → tap **Build Number** 7 times
2. Settings → Developer Options → enable **USB Debugging**
3. Open a terminal app on the device and run:
   ```
   /system/bin/setprop service.adb.tcp.port 5555
   ```
4. In Developer Options, toggle **USB Debugging** OFF then ON (restarts adbd with new port)
5. From your machine: `adb connect <device-ip>:5555`

**Install (first time or update):**
```bash
# First install
adb install app/build/outputs/apk/debug/app-debug.apk

# Update without losing settings
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

## First-Time Setup

1. Open **Magic Mirror Print** from the device app drawer
2. If prompted, tap **Grant Permission** for "Draw over other apps" (needed for manual mode)
3. Tap **Browse** → navigate to the `report` folder on the SD card → tap **Select This Folder**
4. Enter the printer's IP address
5. Confirm **Auto-print** is ON (recommended for this device)
6. Tap **Save Settings**
7. Tap **Start Watcher Service** — the app minimizes automatically
8. Confirm the persistent "Magic Mirror Print Active" notification appears in the status bar

To verify printing works: open the app and tap **Send Test Page**.

The service will restart automatically on every device reboot — no daily setup needed.

## Known Considerations & Future Work

- **SD card path varies** — the folder picker handles this. Common paths on Android 8: `/storage/sdcard1/`, `/storage/extSdCard/`, `/storage/XXXX-XXXX/`
- **CLOSE_WRITE timing** — if the Magic Mirror app triggers multiple close events on the same file, a 1-2 second debounce before printing may be worth adding. Monitor in testing.
- **Print job failure handling** — errors are logged but not retried. A retry queue with toast feedback is a logical next step.
- **Multiple back-to-back scans** — in auto-print mode each scan fires independently. In manual mode the second notification replaces the first (same notification ID).
- **Magic Mirror app updates** — if a future update changes the root directory or report filename, update those values in Settings without needing an app update.
- **kiosk/lock-task mode** — the Magic Mirror app blocks all overlay notifications from third-party apps. Auto-print with sound is the reliable alert mechanism for this device.
