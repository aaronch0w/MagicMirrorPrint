# Magic Mirror Print

An Android sideload app that watches for new client skin analysis reports generated by the **Magic Mirror** medspa device and fires a heads-up print notification — without the staff ever leaving the app or touching the SD card.

---

## The Problem

The Magic Mirror device runs a proprietary Android app that performs multi-camera facial skin analysis. When a scan completes, staff tap "Save to SD Card" which writes a PDF report. To print it, they previously had to:

1. Exit the Magic Mirror app
2. Open the file explorer
3. Navigate to the SD card
4. Find the client's folder
5. Print the PDF

This was too slow and disruptive when a client is sitting in the chair.

## The Solution

Magic Mirror Print runs silently as a **foreground background service**, watching the SD card directory tree for new reports. The moment a new `customer_report.pdf` is written, a **heads-up notification pops over the Magic Mirror app** with two buttons: **Print** and **Dismiss**. Staff tap Print, the job goes to the network printer, and they move on — never leaving the Magic Mirror interface.

## SD Card Directory Structure

The Magic Mirror app writes reports in this exact structure:

```
<root>/                              ← configurable, typically /storage/sdcard1/report/
  └── <Client Name>/                 ← folder named after client entered at scan time
        └── 2026_06_27_23_03_33/    ← date_time folder: YYYY_MM_DD_HH_MM_SS
              ├── img/               ← facial scan images (not our concern)
              └── pdf/
                    └── customer_report.pdf   ← THIS is what we watch for and print
```

**Key behaviors:**
- Each new client gets a new folder under root (named by staff at scan time)
- Each scan session gets a new date/time subfolder under the client folder
- The target file is always named `customer_report.pdf` (configurable in Settings)
- The `CLOSE_WRITE` FileObserver event is used — fires when the writing process closes the file, ensuring the PDF is fully written before we attempt to print

## Architecture

### Files

```
app/src/main/java/com/magicmirrorprint/
├── WatcherService.kt        Foreground service + dynamic recursive FileObserver tree
├── NotificationHelper.kt    Persistent service notification + heads-up report alert
├── PrintManager.kt          Direct IPP/1.1 print over port 631 (no print dialog)
├── AppPrefs.kt              SharedPreferences wrapper — typed getters/setters for all settings
├── SettingsActivity.kt      Main UI: all settings fields, folder picker, start/stop service
├── FolderPickerActivity.kt  Filesystem browser so staff can tap to the report folder at setup
└── BootReceiver.kt          BOOT_COMPLETED receiver — auto-restarts service after device reboot
```

### How the FileObserver tree works

Android's `FileObserver` does not recurse into subdirectories natively. We solve this with a **dynamic observer tree**:

1. **Root observer** watches `<root>/` for new client directories being created → attaches a client observer to each
2. **Client observer** watches `<ClientName>/` for new date/time directories → attaches a date observer to each
3. **Date observer** watches `2026_06_27_23_03_33/` for the `pdf/` subdirectory being created → attaches a PDF dir observer
4. **PDF dir observer** watches `pdf/` for `CLOSE_WRITE` on `customer_report.pdf` → fires the notification

All observers are held in `activeObservers: MutableList<FileObserver>` to prevent garbage collection. Pre-existing folders at service start are also observed (handles the case where service restarts mid-day with clients already scanned).

### Printing (IPP)

Uses raw **IPP/1.1 (Internet Printing Protocol)** via a direct TCP socket on port 631 — no Android Print Framework dialog, no user interaction required. Constructs a minimal `Print-Job` IPP request with:
- `attributes-charset`: utf-8
- `attributes-natural-language`: en
- `printer-uri`: `ipp://<ip>:<port>/ipp/print`
- `document-format`: application/pdf

Runs on a background thread. Compatible with any network printer supporting IPP or AirPrint (HP, Epson, Canon, Brother, etc.).

### Notification flow

```
New customer_report.pdf detected
        │
        ▼
┌─────────────────────────────────────────┐
│  📄 New Report Ready                    │
│  Client: Sarah Johnson                  │
│  Scan: Jun 27, 2026  11:03 PM          │
│                                         │
│  [Print]              [Dismiss]         │
└─────────────────────────────────────────┘
        │                    │
        ▼                    ▼
  IPP print job         Notification
  sent to printer         cleared
```

Two `NotificationChannel`s:
- `channel_watcher` — IMPORTANCE_LOW — persistent "service is running" indicator
- `channel_report` — IMPORTANCE_HIGH — heads-up alert that floats over foreground apps

## Settings

All settings persist in SharedPreferences across restarts.

| Setting | Default | Description |
|---|---|---|
| Root Folder | *(must set)* | Absolute path to the Magic Mirror report directory on SD card. Use the in-app folder browser to set this. |
| Target Filename | `customer_report.pdf` | Filename to watch for. Configurable in case a Magic Mirror update changes it. |
| Printer IP | *(must set)* | Local network IP of the target printer (e.g. `192.168.1.45`) |
| Printer Port | `631` | IPP port — 631 is standard, rarely needs changing |
| Notification Timeout | `60` seconds | How long the heads-up notification stays before auto-dismissing |
| Auto-print | OFF | When ON, prints immediately without showing a notification |

## Build & Deploy

### Requirements
- Android Studio (Hedgehog or later recommended)
- JDK 8+
- ADB installed

### Build
```bash
# Open MagicMirrorPrint/ in Android Studio
# Let Gradle sync
# Build → Build Bundle(s) / APK(s) → Build APK(s)
# APK output: app/build/outputs/apk/debug/app-debug.apk
```

### Sideload via ADB
```bash
# Option A: USB
adb install app/build/outputs/apk/debug/app-debug.apk

# Option B: WiFi ADB (device must be on same network)
adb connect <device-ip>:5555
adb install app/build/outputs/apk/debug/app-debug.apk
```

### Enable ADB on the Magic Mirror device
1. Settings → About → tap **Build Number** 7 times to unlock Developer Options
2. Settings → Developer Options → enable **USB Debugging**
3. For WiFi ADB: also enable **ADB over network** if available, or connect USB first then `adb tcpip 5555`

## First-Time Setup

1. Open **Magic Mirror Print** from the device app drawer
2. Tap **Browse** → navigate to the `report` folder on the SD card → tap **Select This Folder**
3. Enter the printer's IP address (check the printer's network info page or your router's DHCP table)
4. Tap **Save Settings**
5. Tap **Start Watcher Service**
6. Confirm the persistent "Magic Mirror Print Active" notification appears in the status bar

To verify printing works: tap **Send Test Page** in the Settings screen.

## Known Considerations & Future Work

- **SD card path varies by device** — the folder picker handles this. Common paths on Android 8: `/storage/sdcard1/`, `/storage/extSdCard/`, `/storage/XXXX-XXXX/`
- **CLOSE_WRITE timing** — if the Magic Mirror app writes the PDF in a way that triggers multiple close events, we may want to add a 1-2 second debounce before printing. Monitor in testing.
- **Print job failure handling** — current implementation logs errors but does not retry. A retry queue with toast feedback is a logical next improvement.
- **Multiple pending reports** — if two clients are scanned back to back quickly, the second notification replaces the first (same notification ID). Could be changed to stack notifications if needed.
- **Magic Mirror app updates** — if a future update changes the root directory or report filename, update those values in Settings without needing an app update.
